# HTML Element: `<aside>`

The `<aside>` element is used to define content that is tangentially related to the content around it. It's typically used for content like sidebars, pull quotes, advertising, or any content that is related but not the primary focus of the page.

## Syntax

Here's the basic syntax of the `<aside>` element:

```html
<aside>
  <!-- Your aside content goes here -->
</aside>
```

## Attributes

The `<aside>` element can have the following attributes:

### `class`

The `class` attribute is used to add a class to the `<aside>` element, which you can use to style it with [**CSS**](/css).

```html
<aside class="sidebar">
  <!-- Your aside content goes here -->
</aside>
```

### `id`

The `id` attribute is used to add an ID to the `<aside>` element, which you can use to style it with [**CSS**](/css) or target it with [**JavaScript**](/javascript).

```html
<aside id="sidebar">
  <!-- Your aside content goes here -->
</aside>
```

### `style`

The `style` attribute is used to add inline [**CSS**](/css) styles to the `<aside>` element.

```html
<aside
  style="background-color: #f0f0f0; padding: 10px; border: 1px solid #ccc; margin: 10px;"
>
  <!-- Your aside content goes here -->
</aside>
```

### `title`

The `title` attribute is used to add a title to the `<aside>` element, which is displayed as a tooltip when the user hovers over the element.

```html
<aside title="This is the title of the aside element">
  <!-- Your aside content goes here -->
</aside>
```

### `lang`

The `lang` attribute is used to specify the language of the content inside the `<aside>` element.

```html
<aside lang="en">
  <!-- Your aside content goes here -->
</aside>
```

### `dir`

The `dir` attribute defines the text direction (**left-to-right** (`ltr`) or **right-to-left** (`rtl`) ) for the content within the element.

```html
<aside dir="rtl">
  <!-- Your aside content goes here -->
</aside>
```

### `tabindex`

The `tabindex` attribute specifies the tab order of the element. It's typically used to make elements focusable.

```html
<aside tabindex="0">
  <!-- Your aside content goes here -->
</aside>
```

### Example

The below example shows how you can use the `<aside>` element with these attributes:

```html
<aside
  class="sidebar"
  id="important-info"
  style="background-color: #f0f0f0;"
  title="Additional Information"
  lang="en"
  dir="ltr"
  tabindex="1"
>
  Aside with attributes!
</aside>
```

## Real World Scenario Example

Let's say you're writing a blog post about a new JavaScript library you've been working on. You want to include a sidebar with a link to the GitHub repository for the library, as well as a link to the documentation. You also want to include a link to your personal website, where you have a blog post about the library.

Here's how you could use the `<aside>` element to accomplish this:

```html
<article>
  <h1>My New JavaScript Library</h1>
  <p>(content of blog post)</p>
  <aside>
    <h2>Links</h2>
    <ul>
      <li><a href="">GitHub Repository</a></li>
      <li><a href="">Documentation</a></li>
      <li><a href="">Blog Post</a></li>
    </ul>
  </aside>
</article>
```

In this example, the `<aside>` element contains related links that are not the main focus of the article but provide additional context or resources.

You can style the `<aside>` element using [**CSS**](/css), and it can also have attributes like `class` or `id` for further styling or JavaScript interactions.

Here are some examples of how you could style the `<aside>` element:

```css
aside {
  background-color: #f0f0f0;
  padding: 10px;
  border: 1px solid #ccc;
  margin: 10px;
}
```

<Callout type="info" emoji="ℹ️">
    Remember, the use of the `<aside>` element should be **semantically meaningful**. It's _essential_ to use it for content that is related to the main content _but_ not a part of it.
</Callout>
